Basic concepts

According to Google’s official recommendations, the Android testing system should refer to the testing pyramid architecture (as shown in the figure below), and the App should include three types of tests (i.e. small, medium and large tests)：

  • Small tests are unit tests that verify the behavior of an application, one class at a time.
  • Medium tests refer to integration tests that verify interactions between stack levels within a module or between related modules.
  • Large-scale testing refers to end-to-end testing, which is used to verify the user UI click operation process across multiple modules of the application.

Step by step up the pyramid, from small tests to large tests, the fidelity of various tests (the real experience of users) increases step by step, but the execution time and workload required for maintenance and debugging work also increase step by step. Therefore, we should write more unit tests than integration tests, and more integration tests than end-to-end tests. While the proportion of each type of test may vary depending on your application's use cases, we generally recommend the following proportions: 70% for small tests, 20% for medium tests, and 10% for large tests.

Today we are mainly talking about small tests, which account for 70%, also called unit tests.
